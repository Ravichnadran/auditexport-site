Security Model

This document describes how AuditExport executes, accesses data, and enforces security boundaries.

Execution Model

  • AuditExport runs entirely on customer-controlled systems
  • Supported environments include local machines and CI/CD pipelines
  • No SaaS backend or hosted execution environment is used

Data Access

  • Uses read-only access to GitHub APIs
  • Access is provided via environment variables supplied by the user
  • Only repository metadata is collected (no code modification)

Credentials Handling

  • AuditExport does not store credentials
  • Tokens are read only at runtime from environment variables
  • No credentials are written to disk or logs

Data Storage

  • All evidence is written to the local filesystem
  • No data is transmitted to external services
  • Generated artifacts are deterministic and reproducible

Network Usage

  • Outbound connections are limited to GitHub APIs when enabled
  • No inbound network connections are opened

Non-Goals

  • AuditExport is not a certification authority
  • AuditExport does not provide legal or compliance advice
  • AuditExport does not assess organizational compliance status
