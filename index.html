AuditExport

Generate audit-ready technical evidence for ISO/IEC 27001 directly from your GitHub repositories — locally, securely, and deterministically.

What AuditExport does

  • Collects GitHub repository evidence using read-only access
  • Generates immutable JSON evidence and summaries
  • Produces cryptographic hashes for integrity verification
  • Packages outputs into audit-ready ZIP files
  • Runs locally — no SaaS, no data uploads

CLI usage

./auditexport \
  --audit ISO27001 \
  --github \
  --output ./evidence

The command produces deterministic, locally generated audit artifacts suitable for auditor review.

Sample output structure 
evidence/
├── metadata/
│   └── run_metadata.json
├── github/
│   ├── repositories.json
│   ├── branches.json
│   └── contributors.json
├── summary.txt
├── hashes.txt
└── evidence.zip

All outputs are timestamped, immutable, and generated locally.

Frequently asked questions

Is AuditExport a certification tool?

No. AuditExport generates technical evidence only. Compliance decisions remain with auditors.

Does it upload or store data?

No. All evidence is generated locally. Network access is limited to GitHub APIs when enabled.

Can it run in CI/CD?

Yes. AuditExport is designed for CI-based, repeatable evidence generation.

What audits are supported?

v0.1.1 supports ISO/IEC 27001 GitHub evidence collection.
