AuditExport

Generate audit-ready technical evidence for ISO/IEC 27001 directly from your GitHub repositories — locally, deterministically, and without SaaS.

What AuditExport does

  • Collects GitHub repository metadata using read-only access
  • Generates immutable JSON evidence and summaries
  • Produces cryptographic hashes for verification
  • Packages outputs into audit-ready ZIP archives
  • Runs locally or in CI — no data upload

Who AuditExport is for

  • CTOs and Engineering Leads preparing ISO/IEC 27001 audits
  • DevOps and Platform teams responsible for audit evidence
  • Security and compliance engineers
  • SaaS companies onboarding enterprise customers
  • Consultants assisting clients with compliance readiness

What it is — and what it is not

  • Is: A technical evidence generation tool
  • Is: Deterministic, local, repeatable
  • Is not: A certification authority
  • Is not: A SaaS platform
  • Is not: Legal or compliance advice

How it fits into an audit workflow

  • Before audit: Generate evidence snapshots
  • During audit: Share immutable artifacts
  • After audit: Retain evidence

CLI usage example

auditexport \
  --audit ISO27001 \
  --github \
  --output ./evidence

Security & trust model

  • Read-only GitHub tokens
  • No credential storage
  • No external data transmission
