AuditExport Documentation

Practical documentation for generating audit-ready technical evidence using AuditExport.

What AuditExport is (and is not)

AuditExport is a local command-line tool that generates immutable technical evidence for compliance audits.

  • ✔ Generates evidence
  • ✔ Produces repeatable, timestamped outputs
  • ✔ Runs locally or in CI
  • ✘ Does not assess compliance
  • ✘ Does not issue certifications
  • ✘ Does not upload data

Installation

Download the appropriate binary for your platform from the official release page. 

# macOS
unzip auditexport-v0.1.1-macos.zip
chmod +x auditexport

# Linux
unzip auditexport-v0.1.1-linux-x86_64.zip
chmod +x auditexport

Basic usage

./auditexport \
  --audit ISO27001 \
  --github \
  --output ./evidence

This command generates a complete evidence snapshot suitable for auditor review.

CLI options
  • --audit — audit standard identifier
  • --github — enable GitHub evidence collection
  • --output — output directory (default: ./evidence)
  • --dry-run — validate setup without collection
Output structure 
evidence/
├── metadata/
├── github/
├── summary.txt
├── hashes.txt
└── evidence.zip
GitHub access requirements

AuditExport requires a GitHub token with read-only access.

  • No write permissions required
  • No repository modification
  • No token storage on disk
CI/CD usage

AuditExport can be executed in CI pipelines to generate repeatable evidence snapshots.

  • Run on demand or scheduled
  • Store artifacts securely
  • Use for re-certification cycles
Known limitations
  • v0.1.1 supports GitHub only
  • No SaaS integrations
  • No automated compliance scoring

These limitations are intentional to ensure security, predictability, and audit safety.

AuditExport generates technical evidence only. Compliance decisions remain with auditors.
