Documentation

Installation, usage, verification, and audit-safe operation of AuditExport.

Overview

AuditExport generates immutable, audit-ready technical evidence for ISO/IEC 27001 audits. It runs locally or in CI environments and never uploads data to external services.

Supported platforms

  • macOS (Apple Silicon)
  • Linux x86_64 (Ubuntu, Debian, RHEL 8+, Rocky, Alma, Amazon Linux)
  • Windows support planned for a future release

Installation

Linux

unzip auditexport-linux-x86_64.zip
chmod +x auditexport
sudo mv auditexport /usr/local/bin/

macOS

unzip auditexport-macos-arm64.zip
chmod +x auditexport
sudo mv auditexport /usr/local/bin/

Verify Release Integrity

Always verify the downloaded archive before execution.

sha256sum auditexport-linux-x86_64.zip
# Compare with the published .sha256 file

Dry Run (Recommended First Step)

A dry run validates configuration and CLI behavior without collecting any external evidence or requiring credentials. 

./auditexport --audit ISO27001 --github --dry-run
What happens during a dry run:
  • Validates arguments and configuration
  • Creates output directories
  • Does not access GitHub APIs
  • Does not collect or store evidence

GitHub Authentication

AuditExport uses read-only GitHub access via environment variables. Tokens are never stored or transmitted. 

export GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxx
export GITHUB_ORG=your-org-name
# OR
export GITHUB_USER=your-username

Required permissions:

  • repo:read (private repositories)
  • read:org (organization metadata)

Generate ISO/IEC 27001 Evidence

./auditexport --audit ISO27001 --github --output evidence

This command collects GitHub metadata and produces immutable evidence artifacts suitable for audit submission.

Evidence Output Structure

evidence/
├── metadata.json
├── github/
│   ├── repositories.json
│   ├── commits.json
│   └── branches.json
├── checksums.sha256
└── README.txt

All outputs are deterministic, timestamped, and cryptographically verifiable.

CI/CD Example

- name: Generate ISO 27001 audit evidence
  env:
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
  run: |
    ./auditexport --audit ISO27001 --github --output evidence

Common Notes

  • --audit is mandatory for all runs
  • --dry-run still requires --audit
  • Public repositories do not require a token
  • AuditExport never modifies repositories
  • Re-running produces identical output for identical inputs
